# H004_onetrans_anchor — Technical Report

> CLAUDE.md §17.2 anchor (NOT a one-mutation): replaces the PCVRHyFormer
> backbone block with an OneTrans single-stream + mixed-causal block.
> Required during Final Round per the platform notice.

## 1. Hypothesis & Claim
- Hypothesis: **H004_onetrans_anchor**
- Claim: PCVRHyFormer's per-domain encoder + query decoder + RankMixer fusion
  is replaced by a single-stream transformer where S-tokens (4-domain sequence
  events) and NS-tokens (RankMixer chunked feature tokens) share Q/K/V at every
  attention layer. Pushes seq×interaction fusion from block-level (PCVRHyFormer)
  to layer-level (paper arXiv:2510.26104, Tencent UNI-REC).
- Compute tier (suggested): **T2.4 smoke (Taiji)** → T3 full afterwards.
- Expected wall (smoke): ~3–6 min (E_baseline_organizer 2:47 + similar attn cost).

## 2. What this code does
Trains the modified `PCVRHyFormer` wrapper class with `--backbone onetrans`,
which routes the forward pass through the new `OneTransBackbone` instead of
the HyFormer block stack. All sample I/O, dual optimizer (Adagrad sparse +
AdamW dense), embedding tables, classifier head, label_time-aware split, and
10% user OOF holdout are inherited unchanged from H001.

Two patches applied identical to H001 anchor:

- **`label_time`-aware split** — prevents exposure → feedback leakage and
  works on single-row-group parquet files. Toggle with
  `--use_label_time_split`. The H004 anchor `run.sh` keeps the **organizer
  row-group split** (no `--use_label_time_split` flag) so val_AUC pairs
  cleanly with E_baseline_organizer (val_AUC=0.8251).
- **10% user_id OOF holdout** (seed=42) — held out from BOTH train and valid
  so the reported `best_oof_AUC` is a true generalization estimate. Active
  only with `--use_label_time_split`.

Schema is auto-generated by `make_schema.py` if missing. Vocab sizes, list
dims, and per-domain `ts_fid` are inferred from parquet metadata.

## 4. Platform contract
The platform must set:
- `TRAIN_DATA_PATH` — directory containing training `*.parquet`.
- `TRAIN_CKPT_PATH` — writable directory for checkpoints + `metrics.json`.
- (optional) `TRAIN_LOG_PATH`, `TRAIN_TF_EVENTS_PATH` — derive from CKPT if unset.

Then invoke `bash run.sh`. CLAUDE.md §17.8.7: no local-dev fallback — both
`TRAIN_DATA_PATH` and `TRAIN_CKPT_PATH` must be set.

## 5. Outputs
- `${TRAIN_CKPT_PATH}/metrics.json` — primary reporting source. Includes:
  - `best_val_AUC`, `best_oof_AUC` (null if `--use_label_time_split` off)
  - `split_meta`, `seed`, `git_sha`, `config_sha256`, `host`
  - **H004 NEW**: `attn_entropy_per_layer` (list, one entry per OneTrans
    block), `attn_entropy_threshold` (= 0.95 · log(N_tokens)),
    `attn_entropy_violation` (bool).
- `${TRAIN_CKPT_PATH}/global_step*.best_model/` — model.pt + schema.json +
  train_config.json (used by inference).
- `${TRAIN_LOG_PATH}/train.log` — full training log.

## 7. Method extensions over organizer baseline
- **H001 patches inherited**: label_time split (toggleable), user-OOF holdout
  (toggleable), path defaults, auto-generated schema.json, infer.py prior fallback.
- **H004 anchor mutation (new)**: backbone replacement. PCVRHyFormer's
  `MultiSeqHyFormerBlock` + `MultiSeqQueryGenerator` is bypassed; the new
  `OneTransBackbone` builds a single token stream
  `[S_a, S_b, S_c, S_d, NS, CLS]` and runs `--num_onetrans_layers` blocks of
  Pre-LN + multi-head attention (with mixed-causal mask) + FFN. The CLS
  token's final state is fed to the existing `clsfier` head.
- **Mixed-causal mask** (paper §Method, conservative interpretation):
  - S→S causal **within each domain segment**, blocked across domains.
  - S→NS, NS→S, NS→NS bidirectional (layer-level seq×int fusion).
  - CLS row reads everything; CLS column blocked (CLS is sink only).
  - Padding-aware: padded sequence positions blocked as keys.
  - `--mixed_causal_anchor timestamp` reserves the candidate-time filter as
    an extension hook (currently equivalent to `seq_index` because dataset
    pre-filters sequences to events before `label_time`).
- **§10.9 attn entropy diagnostic**: at end of training, one diagnostic batch
  is forwarded through the best model with `log_attn_entropy=True`; per-layer
  mean entropies are written to `metrics.json`. Threshold 0.95 · log(N_tokens)
  is logged alongside; if any layer exceeds, `attn_entropy_violation=True` is
  recorded for verdict carry-forward.

## 8. Reproducibility
- All seeds fixed (default 42); cuDNN deterministic mode set in `utils.set_seed`.
- `--seed` overrides for multi-seed campaigns.
- `metrics.json` records git SHA + config SHA256 for traceability.
- `train_config.json` stores all CLI args (including `backbone`,
  `num_onetrans_layers`, `mixed_causal_anchor`, `domain_id_embedding`); infer.py
  reads them to instantiate the matching backbone for ckpt loading.
